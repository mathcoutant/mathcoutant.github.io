    Laelith Copilot

    My final-year engineering school project involves including a text-generating AI in a role-playing scenario editor. The role-playing game editor already available is for the game called Laelith.

    With our code, the script writer will only have to provide a short description of the action and, if desired, the various results of the extract he's interested in. The AI then writes the text describing the action. This saves the writer a lot of time, and may even lead to some original ideas.

    The main challenge is to restrict the AI to the role-playing universe. It needs to build on existing descriptions and not invent important new elements of the universe while remaining creative.
    We also had to include an interface in an existing project, which meant analyzing the code and rethinking the design of our solution to keep it instinctive.

    Erosion Simulation

    The aim of this project is to create an erosion simulator using shaders. Everything happens in the graphics card, where a compute shader iterates over the terrain to simulate erosion.

    The main difficulty of this project was to work directly on a 3D terrain made up of voxels. Indeed, most existing simulators work on a 2D plane that will be deformed.

    You can find the project on GitHub.

    Wattfor

    This is a school project to assess the power consumption of massively multiplayer video games, in particular Fortnite.

    To do this, we measured the game's power consumption on different configurations. This gives us an estimate of average client-side consumption. To estimate server-side consumption, we wrote a C script simulating the network exchanges that take place between a server and 100 clients.

    The result of this study is that server-side consumption is highly optimized and almost negligible compared to client-side consumption. To reduce the consumption of this type of game, it is therefore essential that client-side games are optimized.
    You can find our report in French here and the simulation code on GitHub.
