    Site personnel

    J'ai codé mon site personnel (celui sur lequel vous êtes) et je l'utilise comme un portfolio.
    Il a été conçu comme un projet d'apprentissage, j'ai décidé de ne pas utiliser de template déjà fait. J'ai donc développé et mis en pratique mes compétences en HTML, CSS et JavaScript pour créer ce site. Je continu de le mettre à jour avec de nouvelles expériences et de nouveaux projets. Et je dois encore l'adapter au format téléphone car il y a de nombreux problèmes de scalling.

    Le code est disponible sur GitHub.

    Laelith Copilot

    Mon projet de dernière année d'école d'ingénieur consiste en l'inclusion d'une IA génératrice de texte dans un éditeur de scénario de jeux de rôle. Le jeu de rôle déjà existant proposant l'éditeur s'appelle Laelith.

    Avec notre ajout, le créateur de scénario aura seulement à fournir une courte description de l'action ainsi que, s'il le veut, les différents résultats de l'extrait qui l'intéresse. Ensuite, l'IA se charge d'écrire le texte qui décrit l'action. Cela permet de faire gagner beaucoup de temps à l'écrivain et peut-être de proposer des idées originales.

    Le principal défi à relever est de restreindre l'IA à l'univers du jeu de rôle. Il faut qu'elle se base sur les descriptions déjà existantes, qu'elle n'invente pas de nouveaux éléments importants de l'univers, le tout en restant créative.
    Il a aussi fallu inclure une interface dans un projet déjà existant, ce qui implique de l'analyse de code et de la reflexion sur le design de notre solution pour que le tout reste instinctif.

    Simulation d'érosion

    L'objectif de ce projet est de créer un simulateur d'érosion en utilisant des shaders. Tout se passe dans la carte graphique où un compute shader itère sur le terrain pour simuler l'érosion.

    La principale difficulté de ce projet était de travailler directement sur un terrain en 3D, composé de voxels. En effet la plupart des simulateurs déjà existants travaillent sur un plan 2D qui va être déformé.

    Vous pouvez retrouver le projet sur GitHub.

    Wattfor

    Il s'agit d'un projet scolaire visant à évaluer la consommation électrique des jeux-vidéos massivement multijoueurs, en particulier de Fortnite.

    Pour cela nous avons mesuré la consommation du jeu sur différentes configurations. Cela nous permet d'avoir une estimation de la consommation moyenne côté client. Pour estimer la consommation côté serveur nous avons écrit un script en C simulant les échanges réseaux qui ont lieu entre un serveur et 100 clients.

    Le résultat de cette étude est que la consommation côté serveur est très optimisée et quasiment négligeable comparé à la consommation côté client. Pour réduire la consommation de ce type de jeu il est donc essentiel que les jeux soient optimisés côté client.
    Vous pouvez retrouver notre rapport ici et le code de la simulation sur GitHub.
